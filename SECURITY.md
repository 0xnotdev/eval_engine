# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✅         |

## Reporting a Vulnerability

If you discover a security vulnerability within AI-Testing-Loops, please report it responsibly.

### How to Report

1. **Do NOT open a public issue.** Security vulnerabilities should be reported privately.
2. **Email:** Send a detailed report to the repository maintainer via GitHub private vulnerability reporting.
3. **Include:**
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment:** Within 48 hours of your report.
- **Assessment:** We will assess the severity and impact within 5 business days.
- **Fix:** Critical vulnerabilities will be patched within 7 days. Non-critical issues will be addressed in the next release cycle.
- **Credit:** We will credit you in the release notes (unless you prefer to remain anonymous).

### Scope

This policy covers the AI-Testing-Loops repository content, including:
- Loop definitions (LOOP.md files)
- Automation scripts (scripts/agent.py)
- CI/CD workflow configurations
- Validation tooling

### Out of Scope

- Third-party tools referenced in loops (e.g., DeepEval, Promptfoo, Garak)
- External APIs and services
- User-generated configurations derived from loop templates

## Responsible Disclosure

We follow [coordinated vulnerability disclosure](https://www.cisa.gov/coordinated-vulnerability-disclosure-process) practices. We ask that you:

- Allow reasonable time for us to address the issue before public disclosure.
- Act in good faith to avoid privacy violations, data destruction, or service disruption.
